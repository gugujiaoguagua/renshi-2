# AI 工作流资产包模板使用说明

如果是第一次使用，建议先读根目录的 `AI工作流资产包使用手册.md`。本文是给 AI 执行时读取的入口说明，使用手册是给人快速上手的操作说明。

## 这个空包解决什么问题

这个模板用于新项目开始时搭建一套可持续成长的 AI 工作流资产包。它不是一次性提示词库，而是把一个项目中的业务规则、执行流程、skills、模板、工具、案例、复盘记录和应用化判断逐步沉淀下来。

目标是让项目从第一天开始就具备三种能力：

1. 一句话启动任务：AI 能根据本包说明读取规则、创建任务卡、选择流程并交付结果。
2. 每次任务后升级：AI 能在任务结束后记录新规则、失败案例、成功格式和技能升级候选。
3. 工作资产可迁移：成熟的流程可以沉淀为 Claude / Codex 可读取的 `SKILL.md` 技能包，或进一步应用化。

## 新项目初始化步骤

1. 复制整个 `AI 工作流资产包模板` 文件夹到新项目目录。
2. 将文件夹重命名为 `AI工作流资产包` 或项目专属名称。
3. 填写 `01_项目说明.md`，说明项目背景、业务对象、当前阶段和资料位置。
4. 填写 `02_角色与目标.md`，定义 AI 在本项目中的角色、边界和交付目标。
5. 检查 `03_规则与约束/00_规则总表.md`，补充本项目必须遵守的硬规则。
6. 用 `06_templates/任务卡模板.md` 创建第一张任务卡，放入 `10_tasks/进行中/`。
7. 执行任务，交付结果后使用 `06_templates/复盘记录模板.md` 写复盘。
8. 按复盘结果更新 `09_memory/`、`03_规则与约束/04_规则升级记录.md`、`05_skills/` 或 `12_app_candidates/`。

## 最小使用法

第一次使用时，不需要维护所有目录。先只用 6 个核心入口：

| 优先级 | 文件或目录 | 用途 |
|---|---|---|
| 必看 | `00_使用说明.md` | 告诉 AI 如何启动本资产包 |
| 必填 | `01_项目说明.md` | 写清楚项目背景和资料位置 |
| 必填 | `02_角色与目标.md` | 写清楚 AI 的角色、边界、目标 |
| 必读 | `03_规则与约束/00_规则总表.md` | 约束 AI 不跑偏 |
| 必用 | `10_tasks/` | 每个任务一张任务卡 |
| 必用 | `09_memory/` | 每次对话、每天、每个阶段的经验沉淀 |

其他目录按需要再用：

- 需要固定流程时，用 `04_流程编排/`。
- 需要复用能力时，用 `05_skills/`。
- 需要标准输出时，用 `06_templates/`。
- 需要脚本工具时，用 `07_tools/`。
- 需要训练样例时，用 `08_examples/`。
- 需要验收时，用 `11_eval/`。
- 需要判断是否做应用时，用 `12_app_candidates/`。

## 给 AI 的启动指令

在新任务开始时，可以直接对 AI 说：

```text
请读取本项目的 AI 工作流资产包，从 00_使用说明.md 开始，按照规则创建任务卡，判断本次任务适合使用已有 skill、创建新 skill 草稿，还是进入应用化需求分析。任务结束后按复盘流程更新资产包。
```

如果是只想执行任务，不希望修改资产包，可以说：

```text
请只读取本项目的 AI 工作流资产包规则来执行任务，本次不要更新资产包文件。
```

## 每次任务的标准流程

1. 读取 `00_使用说明.md`。
2. 读取 `01_项目说明.md` 和 `02_角色与目标.md`。
3. 读取 `03_规则与约束/00_规则总表.md`。
4. 创建或更新任务卡。
5. 判断任务类型：
   - 临时低频任务：直接执行并记录经验。
   - 可复用任务：创建或调用 skill。
   - 高频复杂任务：进入应用化候选。
6. 执行任务并生成交付物。
7. 按 `11_eval/质量检查清单.md` 自检。
8. 写入复盘记录。
9. 更新记忆、规则、技能或应用化候选。

## 每次对话的自动收集机制

每次对话结束前，AI 应自动检查是否出现以下信息：

1. 新业务规则。
2. 用户偏好。
3. 可复用输出格式。
4. AI 误判或容易出错的点。
5. 可沉淀为 skill 的流程。
6. 可加入 examples 的输入输出样例。
7. 高频复杂、可能应用化的需求。

如果有，优先写入 `09_memory/`；只有经过多次验证，才升级到规则总表或 `SKILL.md`。

## 每天结束的总结机制

每天项目工作结束时，使用 `06_templates/每日总结模板.md` 生成当天总结，并更新：

- `09_memory/05_每日总结.md`
- `09_memory/01_业务规则沉淀.md`
- `09_memory/03_失败案例.md`
- `09_memory/04_成功案例.md`
- `09_memory/06_skill成熟度跟踪.md`

每天总结的目的不是写日报，而是判断：今天哪些经验应该保留，哪些 skill 变成熟，哪些任务下一步要继续。

## 项目完成时的收口机制

项目结束时，必须执行 `06_templates/项目完成收口模板.md`：

1. 清点所有任务卡。
2. 清点 memory 中有效经验。
3. 判断哪些规则进入正式规则。
4. 判断哪些 skill 已成熟可导入。
5. 判断哪些流程进入应用化候选。
6. 输出最终可使用的 skill 包清单。

## 经验升级原则

经验不能随便进入正式规则或 `SKILL.md`。采用三层沉淀法：

1. 第一次出现：记录到 `09_memory/` 或任务复盘。
2. 第二次验证：加入规则候选或 skill 候选。
3. 第三次稳定：升级到规则总表、流程文件或 `SKILL.md`。

## 技能包与应用化判断

优先做 skill，不要一开始就做应用。

适合做 skill：

- 流程清晰但不需要界面。
- 一个人或少数人使用。
- 输入输出以文件、文本、表格为主。
- 需要 AI 根据规则完成判断、整理、生成、分析。

适合应用化：

- 多人高频使用。
- 需要登录、权限、状态流转或数据长期保存。
- 需要手机端、网页端或固定操作界面。
- skill 已经跑通，但人工启动和整理成本过高。

## 第一版验收口径

这个模板第一版验收时，只看是否具备以下能力：

1. 新项目能按说明初始化。
2. 新任务能按任务卡启动。
3. 有规则总表和规则升级记录。
4. 有流程编排文件。
5. 有可导入 skill 的标准目录和示例 `SKILL.md`。
6. 有复盘模板和 memory 目录。
7. 有质量检查和应用化候选判断。
