# 目录分级与最小使用法

## 检查结论

当前目录不需要再扩张。第一版目录覆盖了工作流资产包的完整生命周期，但新项目启动时不应该全部维护，否则会显得繁琐。

使用原则：

```text
先用核心目录跑通，再按需要启用扩展目录。
```

## 核心目录

这些是每个项目都要用的。

| 目录或文件 | 必要性 | 用途 |
|---|---|---|
| `00_使用说明.md` | 必用 | AI 启动说明 |
| `01_项目说明.md` | 必填 | 项目背景和资料位置 |
| `02_角色与目标.md` | 必填 | AI 角色和执行边界 |
| `03_规则与约束/` | 必用 | 项目硬规则、执行边界、规则升级 |
| `10_tasks/` | 必用 | 每个任务的任务卡 |
| `09_memory/` | 必用 | 每次对话、每天、项目阶段的经验沉淀 |

## 半核心目录

这些在项目开始后很快会用到，但不是第一分钟必须填满。

| 目录 | 什么时候用 |
|---|---|
| `04_流程编排/` | 当任务开始重复、需要固定执行顺序 |
| `06_templates/` | 当输出格式要稳定复用 |
| `11_eval/` | 当需要验收任务、skill 或项目阶段 |

## 扩展目录

这些按成熟度启用。

| 目录 | 什么时候用 |
|---|---|
| `05_skills/` | 当某类任务重复出现，值得沉淀为技能 |
| `07_tools/` | 当有脚本、API、自动化工具 |
| `08_examples/` | 当有典型输入输出样例 |
| `12_app_candidates/` | 当流程高频、多人使用、需要界面或数据持久化 |
| `99_archive/` | 当资料过期或废弃 |

## 最小启动指令

```text
请读取本项目的 AI 工作流资产包。先只使用核心目录：00_使用说明.md、01_项目说明.md、02_角色与目标.md、03_规则与约束、10_tasks、09_memory。请为本次任务创建任务卡，任务结束后收集有用信息并更新 memory。
```

## 完整启动指令

```text
请读取本项目的 AI 工作流资产包，按完整流程执行：创建任务卡、判断任务类型、调用或创建 skill、完成交付、质量检查、复盘、更新 memory/rules/skills/app candidates。
```
