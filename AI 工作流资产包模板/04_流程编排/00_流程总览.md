# 流程总览

## 总流程

```text
任务进入
→ 读取使用说明、项目说明、角色目标、规则总表
→ 创建任务卡
→ 判断任务类型
→ 选择执行路径
→ 执行并交付
→ 质量检查
→ 对话结束信息收集
→ 复盘
→ 更新 memory / rules / skills / app candidates
→ 每日总结或项目收口
```

## 三条执行路径

### 路径 A：临时任务

适合低频、一次性、规则不稳定的任务。

动作：

1. 创建任务卡。
2. 直接完成交付。
3. 复盘时记录经验，不创建正式 skill。

### 路径 B：skill 任务

适合流程清晰、可复用、主要由 AI 执行的任务。

动作：

1. 查找已有 skill。
2. 没有则创建 skill 草稿。
3. 用真实任务验证。
4. 复盘后更新 examples、references 或升级记录。

### 路径 C：应用化任务

适合多人高频、需要界面、权限、数据持久化或状态流转的任务。

动作：

1. 先用 skill 跑通流程。
2. 进入 `12_app_candidates/应用化候选清单.md`。
3. 补充用户、流程、数据、权限、验收标准。
4. 读取 `03_规则与约束/05_AI友好代码架构规范.md`。
5. 使用 `06_templates/代码项目架构模板.md` 确定代码结构。
6. 再决定是否搭建 demo。

### 路径 D：代码开发任务

适合已经决定搭建 demo、小工具或应用的任务。

动作：

1. 创建任务卡。
2. 读取 AI 友好代码架构规范。
3. 确定项目类型：轻量工作流 / 前端应用 / 全栈应用。
4. 填写代码项目架构模板。
5. 先搭最小 demo。
6. 投入真实使用。
7. 根据使用反馈更新代码、skill、规则和应用化候选。

## 自动迭代闭环

资产包的升级不是靠一次性整理，而是靠固定闭环：

```text
每次对话结束
→ 收集有用信息
→ 写入 memory
→ 每天结束做总结
→ 多次验证后升级规则或 skill
→ 项目完成时发布成熟 skill 包
```

## 升级优先级

| 信息类型 | 第一次出现 | 多次验证后 |
|---|---|---|
| 业务规则 | `09_memory/01_业务规则沉淀.md` | `03_规则与约束/00_规则总表.md` 或 skill references |
| 用户偏好 | `09_memory/02_用户偏好.md` | `02_角色与目标.md` 或使用说明 |
| 失败案例 | `09_memory/03_失败案例.md` | `11_eval/常见错误清单.md` |
| 成功案例 | `09_memory/04_成功案例.md` | `08_examples/输出样例/` |
| 可复用流程 | 任务复盘 | `05_skills/` |
| 高频复杂流程 | 任务复盘 | `12_app_candidates/应用化候选清单.md` |
