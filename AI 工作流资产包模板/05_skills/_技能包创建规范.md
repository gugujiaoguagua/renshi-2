# 技能包创建规范

## 技能包定位

skill 用来承载某一类可复用任务的执行规则。它应该让 AI 在相近业务场景中知道怎么开始、怎么判断、怎么输出、怎么验收。

## 标准目录

```text
某个技能/
├─ SKILL.md
├─ references/
├─ examples/
└─ scripts/
```

## SKILL.md 应包含

- frontmatter：`name` 和 `description`
- 触发场景
- 不适用场景
- 输入材料
- 执行流程
- 输出格式
- 质量检查
- 复盘升级规则

## SKILL.md 不应包含

- 大量历史聊天记录
- 临时讨论
- 未验证的一次性规则
- 大段业务资料全文
- 与技能无关的背景

## references 放什么

- 业务术语
- 字段说明
- 判断规则
- 流程细节
- 长文档摘要

## examples 放什么

- 标准输入样例
- 标准输出样例
- 失败样例
- 边界情况样例

## scripts 放什么

- 可重复执行的脚本
- 文件整理工具
- Excel 清洗工具
- OCR 批处理工具
- 报告生成工具

## 成熟度等级

| 等级 | 名称 | 标准 |
|---|---|---|
| L0 | 临时提示词 | 只解决一次任务 |
| L1 | skill 草稿 | 有初步流程 |
| L2 | 可复用 skill | 被 2 次以上任务验证 |
| L3 | 可导入 skill | 有标准 `SKILL.md` |
| L4 | 带样例 skill | 有 examples 和质量检查 |
| L5 | 应用化候选 | 高频多人使用，值得做 demo |
