# 示例业务流程技能

## 适用场景

用于演示一个可导入 Claude / Codex 的 skill 应该如何组织。复制本目录后，将名称、描述、流程和输出格式替换为真实业务场景。

## 不适用场景

- 用户只想闲聊或讨论方向。
- 输入材料不足且无法判断。
- 任务涉及高风险决策，需要负责人确认。
- 已经明显需要完整应用系统，而不是单次 AI 执行流程。

## 输入材料

启动前检查：

1. 用户目标。
2. 原始材料路径。
3. 期望输出格式。
4. 业务规则或参考样例。
5. 是否允许更新资产包。

## 执行流程

1. 读取项目规则。
2. 建立任务卡。
3. 检查输入材料是否足够。
4. 按业务流程处理材料。
5. 生成交付物。
6. 使用质量检查清单自检。
7. 任务结束后写复盘建议。

## 输出格式

默认输出：

```text
任务结论：
关键发现：
生成文件：
不确定项：
下一步建议：
资产包升级建议：
```

## 质量检查

- 是否满足用户目标。
- 是否引用真实输入。
- 是否标明不确定项。
- 是否未编造业务规则。
- 是否给出可执行下一步。

## 升级规则

新经验先写入 memory 或复盘。只有经过多次验证后，才升级到本 `SKILL.md`。
