适合脚本、报告生成、自动化处理。

```text

├─ docs/
├─ scripts/
├─ samples/
├─ outputs/
└─ README.md
```

### 应用项目结构

适合 Web / App / 后台系统。

```text

├─ app/
│  ├─ core/
│  ├─ shared/
│  ├─ features/
│  └─ layouts/
├─ assets/
└─ environments/
```

## AI 修改边界

本项目要求 AI 修改代码前先确认：

1. 本次修改属于哪个模块。
2. 是否需要先更新 types。
3. 是否需要先更新 rules。
4. 是否需要新增或更新 services。
5. 是否需要更新页面。
6. 是否需要补测试或验证样例。

## 验收清单

- [ ] 目录按业务模块组织。
- [ ] 核心模块有 README。
- [ ] 业务规则没有写死在 UI 中。
- [ ] 数据访问已封装在服务层。
- [ ] 新字段已先更新类型定义。
- [ ] 关键规则有测试或验证样例。
- [ ] 代码改动范围与任务卡一致。
