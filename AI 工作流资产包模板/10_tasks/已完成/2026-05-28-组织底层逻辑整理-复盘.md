# 复盘：组织底层逻辑整理

## 结论

组织域已经先行归拢，不再只是几个接口各自取数。现在组织结构、岗位、职级、组织设置有统一快照，员工模块后续应以这个快照作为上游口径。

## 做对的地方

- 没有新增零散业务文件，组织逻辑仍集中在 `data-server.mjs` 的组织区块内。
- summary 和 foundation 共用同一份组织快照，避免统计对不上。
- 新接口只服务后端底层检查，不影响前端 UI。
- API smoke test 同时覆盖新接口和原组织列表接口，确认原功能没有断。

## 风险和注意

- 组织基础检查目前是轻量质量检查，后续如果发现真实数据规则更多，可以继续补到同一个快照里。
- 员工整理时不能绕过组织快照直接读散落字段，否则又会回到口径不一致。

## 下一步建议

继续按主线进入员工：先整理员工主数据来源、组织/岗位/职级关联规则、员工输出给考勤的字段契约。
