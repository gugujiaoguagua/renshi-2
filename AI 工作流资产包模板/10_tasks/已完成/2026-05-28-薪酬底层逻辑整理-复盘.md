# 复盘：薪酬底层逻辑整理

## 结论

薪酬域已经接到考勤之后，形成了统一快照。后端现在明确了薪酬只能消费员工主数据、考勤月汇总、统计项和外部数据，不能绕过考勤域直接读取原始打卡明细。

## 做对的地方

- 没有改 UI，也没有迁移前端本地薪酬计算，避免扩大范围。
- 先固定薪酬数据边界和公式追踪结构，为后续迁移计算做准备。
- API smoke test 同时覆盖薪酬、员工、考勤、月汇总、统计项和外部数据，确认四段主线能串起来。
- 明确当前薪酬核算页仍有本地样例和本地公式，后续迁移不能绕过本次定义的后端输入边界。

## 风险和注意

- 本次没有实现新的后端薪酬计算接口，只整理底层口径。
- 外部数据当前为 0，后续如果导入外部统计项，需要继续验证员工号、期间、统计项三者是否唯一。
- 前端 `SalaryCalculation.tsx` 仍有本地计算逻辑，后续应按 `payrollDomainSnapshot()` 的输入边界逐步迁移。

## 下一步建议

做四段主线总体验收，并生成一份后端底层逻辑说明：组织、员工、考勤、薪酬分别有哪些输入、输出、写入规则、质量检查和接口。
