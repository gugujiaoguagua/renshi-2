# 验收标准

## 项目级验收

一个 AI 工作流资产包合格，应满足：

1. 有项目说明和角色目标。
2. 有规则总表。
3. 有任务启动、执行、验收、复盘流程。
4. 有任务卡模板和复盘模板。
5. 有 memory 记录区。
6. 有 skill 标准目录。
7. 有质量检查清单。
8. 有应用化候选判断。
9. 有对话结束信息收集机制。
10. 有每日总结机制。
11. 有项目完成收口机制。

## 任务级验收

一个任务合格，应满足：

1. 有明确任务目标。
2. 有输入材料记录。
3. 有交付结果。
4. 有不确定项说明。
5. 有质量检查。
6. 有复盘或复盘建议。

## skill 级验收

一个 skill 合格，应满足：

1. 有 `SKILL.md`。
2. frontmatter 有 `name` 和 `description`。
3. description 能准确说明触发场景。
4. 主体流程清晰。
5. 输出格式稳定。
6. 有升级规则。

## 项目完成验收

一个项目结束时，应满足：

1. 所有任务卡已完成、暂停或归档。
2. 有项目完成收口记录。
3. 有最终规则清单。
4. 有最终 skill 清单。
5. 至少一个成熟 skill 达到 L3，或明确说明为什么没有达到。
6. 成熟 skill 有 `SKILL.md`、输入说明、输出格式和质量检查。
7. 典型成功案例进入 examples 或 memory。
8. 高频复杂流程进入应用化候选池。

## 代码架构验收

当项目进入 demo 或应用开发时，应满足：

1. 已读取 `03_规则与约束/05_AI友好代码架构规范.md`。
2. 已使用 `06_templates/代码项目架构模板.md` 确定项目结构。
3. 代码目录按业务模块组织。
4. 核心模块有 README。
5. 业务规则没有写死在 UI 事件中。
6. 数据访问已封装在 services/api 层。
7. 新字段先更新 types。
8. AI 修改代码前能明确修改范围和验证方式。
