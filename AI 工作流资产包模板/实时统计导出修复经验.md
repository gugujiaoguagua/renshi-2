# 实时统计导出修复经验

更新时间：2026-06-01

适用范围：`D:\我的应用\打卡前+后\后端` 中考勤管理、薪酬核算等表格页面的“勾选记录后导出 Excel”问题。

## 这次问题

【实时统计】页面最初有两个导出相关问题：

1. 表头选择框无法全选。
2. 导出按钮不像【薪酬核算 / 全部岗位】那样弹出选择保存位置，也不是同样的红色主按钮样式。

本质上是两个链路没有对齐：

1. 表格选择链路：表头和行 checkbox 只是静态 `<input type="checkbox" />`，没有 `checked`、`onChange`、半选状态和选中集合。
2. 导出保存链路：实时统计使用普通下载逻辑，薪酬核算使用 `showSaveFilePicker` 让用户选择保存位置。

## 判断方法

遇到“无法选择导出”“导出不是选中数据”“全选无效”时，先不要直接改导出函数，按这个顺序查：

1. 查表头 checkbox 是否绑定状态。
   - 应该有 `checked={allPageSelected}`。
   - 应该有 `onChange={togglePageSelection}`。
   - 部分选中时应设置 `el.indeterminate = true`。

2. 查行 checkbox 是否绑定同一个选择集合。
   - 应该有 `checked={selectedRowIds.has(rowId)}`。
   - 应该有 `onChange={() => toggleRowSelection(rowId)}`。

3. 查导出数据来源。
   - 如果有选中数据，导出 `selectedRows`。
   - 如果没有选中数据，导出当前筛选结果。
   - 不要只导出当前分页，除非需求明确说“只导出本页”。

4. 查保存方式。
   - 如果用户要求“像全部岗位一样”，需要走 `showSaveFilePicker`。
   - 只用 `XLSX.writeFile(...)` 或创建 `<a download>`，一般是浏览器默认下载，不会弹出选择保存位置。

## 关键代码模式

实时统计页面在 `src/app/pages/AttendanceStats.tsx` 中补了选择状态：

```tsx
const [selectedRowIds, setSelectedRowIds] = useState<Set<string>>(new Set());

const getRowId = (emp: Employee) => emp.empId || `${emp.name}-${emp.dept}-${emp.attendGroup}`;
const pageRowIds = pagedEmployees.map(getRowId);
const allPageSelected = pageRowIds.length > 0 && pageRowIds.every(id => selectedRowIds.has(id));
const somePageSelected = pageRowIds.some(id => selectedRowIds.has(id)) && !allPageSelected;

const togglePageSelection = () => {
  setSelectedRowIds(current => {
    const next = new Set(current);
    if (allPageSelected) pageRowIds.forEach(id => next.delete(id));
    else pageRowIds.forEach(id => next.add(id));
    return next;
  });
};

const toggleRowSelection = (rowId: string) => {
  setSelectedRowIds(current => {
    const next = new Set(current);
    if (next.has(rowId)) next.delete(rowId);
    else next.add(rowId);
    return next;
  });
};
```

表头 checkbox 用这个模式：

```tsx
<input
  type="checkbox"
  checked={allPageSelected}
  ref={el => { if (el) el.indeterminate = somePageSelected; }}
  onChange={togglePageSelection}
/>
```

行 checkbox 用这个模式：

```tsx
<input
  type="checkbox"
  checked={selected}
  onChange={() => toggleRowSelection(rowId)}
/>
```

导出时按“选中优先，否则当前筛选结果”：

```tsx
const selectedRows = selectedRowIds.size
  ? sortedEmployees.filter(emp => selectedRowIds.has(getRowId(emp)))
  : sortedEmployees;
```

## 选择保存位置的导出

这次抽了通用工具：`src/app/shared/export/attendanceExport.ts`。

普通导出：

```tsx
downloadAttendanceXlsx({
  fileName: '某页面-筛选结果.xlsx',
  sheetName: '某页面',
  headers,
  rows,
});
```

像【全部岗位】一样选择保存位置：

```tsx
downloadAttendanceXlsx({
  fileName: '实时考勤统计-筛选结果.xlsx',
  sheetName: '实时统计',
  headers,
  rows,
  saveAs: true,
});
```

`saveAs: true` 会走 `window.showSaveFilePicker`。如果当前浏览器不支持，会提示“当前浏览器不支持选择保存位置”。

## 按钮样式对齐

【实时统计】按钮要对齐【全部岗位】红色主按钮，不要用灰色描边按钮。样式核心是：

```tsx
style={{
  height: 32,
  border: 'none',
  borderRadius: 4,
  background: exportDisabled ? colors.inputBorder : colors.primary,
  color: exportDisabled ? colors.textMuted : colors.primaryText,
  display: 'inline-flex',
  alignItems: 'center',
  gap: 6,
  padding: '0 10px',
  cursor: exportDisabled ? 'not-allowed' : 'pointer',
  fontSize: 13,
  fontWeight: 600,
  whiteSpace: 'nowrap',
}}
```

图标用：

```tsx
<Download size={14}/>导出Excel
```

## 下次修复同类页面的清单

1. 先确认用户说的“无法选择”是“无法勾选记录”，还是“无法选择保存位置”。
2. 如果是无法勾选，先补 `selectedRowIds`、表头全选、行选择、半选状态。
3. 如果是无法选择保存位置，检查导出工具是否支持 `saveAs: true`。
4. 如果要像【全部岗位】，按钮样式使用红色主按钮，不使用 `outlineBtn`。
5. 导出数据源必须明确：
   - 有勾选：导出勾选记录。
   - 无勾选：导出当前筛选后的全部记录。
6. 修改后跑：

```bash
npm run build
```

构建通过后，再在页面刷新验证：

1. 表头全选是否能选中当前页。
2. 部分选中时表头是否半选。
3. 勾选几条后导出文件名是否带“选中记录”。
4. 不勾选时是否导出“筛选结果”。
5. `saveAs: true` 的页面是否弹出保存位置选择框。
